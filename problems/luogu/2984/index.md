### 题意

给一张带正边权的无向图，`1` 号牧场是仓库。

每个询问给出一头公牛所在的牧场 `p`，以及它想送巧克力的母牛所在牧场 `q`。

这头公牛必须先到 `1` 号牧场拿巧克力，再从 `1` 号牧场走到 `q`。

要求输出每个询问的最短总路程。

### 思路

先看一个最直接的小数据暴力：

@include-code(./brute.cpp, cpp)

暴力做法是 Floyd：

1. 先求任意两点最短路
2. 每个询问直接输出 `dist[p][1] + dist[1][q]`

这个做法很好理解，但这题真正的关键观察非常短：

- 路线被强制拆成 `p -> 1` 和 `1 -> q`

也就是说，询问之间的公共部分就是：

- 所有答案都要用到“从 1 号点到其他所有点的最短路”

于是只要：

1. 从 `1` 号点跑一次 Dijkstra
2. 记下 `dist[x]` 表示 `1 -> x` 的最短距离
3. 对每个询问输出 `dist[p] + dist[q]`

因为图是无向图，所以：

- `dist(p, 1) = dist(1, p)`
- `dist(1, q)` 也已经在同一次最短路里求出来了

所以整题只需要一次单源最短路。

### 复杂度

一次 Dijkstra：

- `O((N + M) log N)`

回答 `B` 个询问只要顺序输出：

- `O(B)`

总复杂度：

- `O((N + M) log N + B)`

空间复杂度：

- `O(N + M)`

### 总结

这题是一个非常标准的“先看清路径被什么条件强制拆开”的题。

一旦发现每条路线都必须经过 `1`，问题就不再是“很多次最短路查询”，而是：

- 先从 `1` 跑一次单源最短路
- 再把每个询问拆成两段距离直接相加

所以本质上仍然是单源最短路模板题。
