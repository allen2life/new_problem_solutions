### 题意

给出区间 `[a,b]`，要求输出其中所有既是回文数、又是质数的整数，每个数单独一行。

### 思路

先看一个可以直接验证想法的朴素解：

从 `a` 到 `b` 逐个枚举每个整数：

1. 判断它是不是回文；
2. 如果是，再判断它是不是质数；
3. 两个条件都满足就输出。

@include-code(./brute.cpp, cpp)

这个做法的问题是：区间可能非常大，但真正是回文数的整数很少。

所以更好的办法不是在区间里“找回文”，而是直接“造回文”。

这里最关键的性质是：

- 除了 `11` 以外，所有偶数位回文数都能被 `11` 整除，因此不可能是质数。

这就意味着真正需要检查的候选数只有两类：

1. `11`
2. 所有奇数位回文数

奇数位回文数可以由一个种子直接构造出来：

| 种子 | 构造结果 |
| --- | --- |
| `7` | `7` |
| `12` | `121` |
| `305` | `30503` |

表格左列表示“前半部分”，右列表示把它镜像后得到的奇数位回文数。  
例如种子 `12`，保留最后一位 `2` 作为中心，再把前面的 `1` 逆序补回去，就得到 `121`。

于是正式做法就是：

1. 处理一位回文数；
2. 单独特判 `11`；
3. 从小到大枚举种子，构造奇数位回文数；
4. 对落在区间里的候选数做试除法判素；
5. 是质数就输出。

### 复杂度

正式做法只检查大约 `10^4` 级别的回文候选数，每个候选数用试除法判素。  
在本题范围内，这个复杂度完全足够。空间复杂度是 `O(1)`。

### 总结

这题的核心不是判素数，而是减少候选数。

一旦看出“除了 `11` 外，偶数位回文数都不可能是质数”，问题就从“区间暴力”变成了“构造少量候选 + 判素”，复杂度会立刻降下来。
