### 题意

给出 `k` 组询问，每组有三个正整数 `n,e,d`。

要求寻找正整数 `p,q`，满足：

```text
n = p * q
e * d = (p - 1)(q - 1) + 1
```

如果存在这样的 `p,q`，输出它们；否则输出 `NO`。

### 思路

先看一个可以直接验证想法的朴素解：

@include-code(./brute.cpp, cpp)

朴素做法枚举 `p`，如果 `p` 能整除 `n`，就令 `q=n/p` 再检查第二个式子。这个思路正确，但 `n` 可以到 `10^18`，即使枚举到 `sqrt(n)` 也太慢。

关键是把第二个式子展开：

| 式子 | 含义 |
| --- | --- |
| `e*d = (p-1)(q-1)+1` | 题目给出的条件 |
| `e*d = pq - p - q + 2` | 展开括号 |
| `e*d = n - p - q + 2` | 因为 `pq=n` |
| `p+q = n - e*d + 2` | 得到两个数的和 |

令：

```text
sum = p + q = n - e*d + 2
```

现在我们已经知道了 `p*q=n` 和 `p+q=sum`。这说明 `p,q` 是方程：

```text
x^2 - sum*x + n = 0
```

的两个根。

所以只需要判断这个二次方程有没有正整数根。判别式为：

```text
delta = sum^2 - 4*n
```

若 `delta` 不是非负完全平方数，就没有整数解。若 `root = sqrt(delta)`，则两个根只能是：

```text
p = (sum - root) / 2
q = (sum + root) / 2
```

这里的 `sqrt(delta)` 需要求整数平方根。rbook《二分查找》文章中讲到，二分可以用来在单调空间中找边界；本题就是二分最大的 `x`，使得 `x*x <= delta`。

### 复杂度

每组询问二分一次整数平方根，平方根范围不超过 `10^9`。

- 单组时间复杂度 `O(log 10^9)`。
- 总时间复杂度 `O(k log 10^9)`。
- 空间复杂度 `O(1)`。

### 总结

这题看起来是在分解 `n`，但真正的突破口是把两个条件合起来推出 `p+q`。

已知两数的和与积，就可以转成一元二次方程。之后只要检查判别式是不是完全平方数，以及根是不是正整数即可。二分在这里不是二分答案，而是用来稳定地求整数平方根，避免浮点误差。
