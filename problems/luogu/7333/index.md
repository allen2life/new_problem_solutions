### 题意

给定一个有 `n` 个点的环，相邻点距离为 `1`。

每个点 `i` 有两个属性 `a_i` 和 `b_i`。  
要求对每个点 `i`，找出满足：

- `j != i`
- `a_j >= b_i`

的点 `j` 中，和 `i` 在环上的最短距离。

如果不存在这样的 `j`，输出 `-1`。

### 思路

暴力做法就是对每个 `i` 枚举所有 `j`，检查 `a_j >= b_i`，然后更新最小环距离。复杂度是 `O(n^2)`，过不了 `10^5`。

这题的关键是把“满足 `a_j >= b_i` 的点”看成一个按阈值变化的集合。

如果我们把所有点按 `a_j` 从大到小排序，再把所有询问点按 `b_i` 从大到小排序，那么在处理某个 `b_i` 时：

- 所有 `a_j >= b_i` 的点都应该已经加入集合
- 所有 `a_j < b_i` 的点都还没加入集合

于是问题就变成：

> 在一个环上，给定若干个“活跃位置”，求某个位置到最近活跃位置的距离。

我们用一个有序集合 `set<int>` 维护当前所有活跃位置。  
对于位置 `i`，离它最近的活跃点只可能是：

- 在有序集合里紧靠它右边的点
- 紧靠它左边的点
- 以及考虑环形首尾相连后，集合里的最小点和最大点

检查这几个候选位置，取最小环距离即可。

注意还要排除 `j = i` 的情况：如果集合里只有自己一个点，答案应为 `-1`；如果最近位置恰好等于自己，就继续看前驱/后继。

整个过程是典型的离线排序 + 有序集合维护。

### 复杂度

- 排序复杂度：`O(n log n)`
- 每个点插入集合一次、查询一次：`O(n log n)`

总时间复杂度是 `O(n log n)`，空间复杂度是 `O(n)`。

### 总结

这题的核心不是环，而是先看出“`a_j >= b_i` 可以按阈值离线处理”。

离线之后，环上的最近点查询只需要在有序集合里看前驱后继，再补上首尾跨环情况即可。
