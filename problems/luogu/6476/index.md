### 题意

有无限多个格子。

- `p1` 的倍数必须染红
- `p2` 的倍数必须染蓝
- 同时是两者倍数的位置可以任选红或蓝

忽略未染色格子后，如果出现 `k` 个连续格子颜色相同，就算“无聊”。

题目要求判断：是否存在一种染色方案，使它不是无聊的。

### 思路

先看一个可以直接验证想法的小数据暴力：

@include-code(./brute.cpp, cpp)

`brute.cpp` 会把一个周期内的必染色类型抽出来，然后在状态图上判断是否存在无限延续的合法染色方案。

这个做法适合小数据理解，但显然不可能应对 `10^6` 组、`10^9` 量级的数据。

#### 第一步：先除掉 gcd

设：

- `g = gcd(p1, p2)`

把所有下标都除以 `g`，不会改变“染色顺序里相邻关系”的结构，所以只需要研究：

- `a = p1 / g`
- `b = p2 / g`

并设 `a <= b`。

这样就有：

- `gcd(a, b) = 1`

#### 第二步：谁更容易形成长连续段？

因为 `a <= b`，所以：

- `a` 的倍数更密
- `b` 的倍数更稀

直觉上，真正危险的是“很多个 `a` 的倍数连续出现”，因为稀疏的那一色根本来不及把它们打断。

如果我们把所有公共点都染成较稀疏的那一色，那么蓝色连续段最多只会很短，问题就转化成：

> 在相邻两个 `b` 的倍数之间，最多会出现多少个只能染成红色的点？

#### 第三步：强制红段长度是多少？

看相邻两个 `b` 的倍数：

- `jb`
- `(j+1)b`

在这两个点之间，所有被染色的内部点都只能是 `a` 的倍数。  
因为 `gcd(a,b)=1`，内部不会再出现公共点来帮你切开这段红色。

所以这中间会形成一整段连续红色。

这段长度的最大值是：

`ceil((b - 1) / a)`

用整数写法就是：

`(a + b - 2) / a`

#### 第四步：判定条件

如果：

- `k` 大于这个最大强制同色段长度

那么就可以通过把公共点尽量染成较稀疏的颜色，成功避免出现长度为 `k` 的同色连续段。

否则，无论怎么染，都会被某一段强制红色卡住。

所以答案就是：

- 若 `k > ceil((b - 1) / a)`，输出 `Yes`
- 否则输出 `No`

#### 特殊情况

如果 `p1 = p2`，那么所有必须染色的位置其实都是公共点，可以随意交替染色。

这时只要 `k > 1`，就一定有解。

### 复杂度

- 时间复杂度：每组 `O(log min(p1,p2))`
- 空间复杂度：`O(1)`

### 总结

这题的关键不是模拟染色，而是先做两个转化：

1. 用 `gcd` 把问题归一化
2. 把“是否无聊”转成“强制同色段的最大长度是多少”

一旦发现真正的瓶颈是相邻两个稀疏倍数之间的那段连续稠密颜色，题目就只剩一个简单公式判定。
