### 题意

我们从 `(1,1)` 出发，要走到 `(M,M)`。

每次只能去右下方向的岛，也就是下一个岛必须满足：

- `x` 不减
- `y` 不减

走到一个岛能拿到这座岛的水果价值，两个岛之间的划船代价是横纵坐标差平方和。

要求最大化：

`拿到的水果总价值 - 全部划船代价`

### 思路

先看最直接的暴力 DP：

@include-code(./brute.cpp, cpp)

设 `dp(i)` 表示到达岛 `i` 的最大收益。

如果 `j` 能转移到 `i`，那么：

`dp(i) = max(dp(j) + v_i - (x_i-x_j)^2 - (y_i-y_j)^2)`

把平方项展开：

`dp(i) = v_i - x_i^2 - y_i^2 + max(dp(j) - x_j^2 - y_j^2 + 2x_ix_j + 2y_iy_j)`

问题在于前驱要同时满足：

- `x_j <= x_i`
- `y_j <= y_i`

这是一个二维偏序约束。

这时利用 `M <= 1000` 这个条件，可以按列 `x` 从小到大扫描。

对于历史上的某个点 `(x_j, y_j)`，如果把 `y_j` 固定住，那么它对未来列 `x` 的贡献可以写成一条直线：

`f_j(x) = 2x_j * x + (dp(j) - x_j^2 - y_j^2)`

所以我们对每一行 `y` 维护一个凸壳，表示“最后一个点落在这一行”的所有历史状态。

扫描到当前列 `x` 时：

1. 先对每一行查询出在当前 `x` 处的最优值 `g[y]`
2. 再在当前列内部按 `y` 从小到大处理

当前列内部还要保证前驱的 `y_j <= y_i`，所以再维护两个凸壳：

- `prefix_hull`：收集所有 `y' <= y` 的历史行结果
- `same_column_hull`：收集当前列里更上方点的结果

这样就把二维偏序 DP 拆成了两次单调凸包优化。

### 复杂度

去重后的岛数记为 `K`。

时间复杂度 `O(M^2 + K)`，空间复杂度 `O(M^2 + K)`。

### 总结

这题的难点不是状态定义，而是怎么利用“只能往右下走”的偏序约束。

把式子展开后，可以看出它本质是：

- 历史列对当前列的直线查询
- 当前列内部对纵坐标的前缀最优转移

一旦把这两层拆开，代码结构就清楚了。
