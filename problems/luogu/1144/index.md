### 题意

给定一张无向无权图，求从 `1` 号点到每个点的最短路条数，对 `100003` 取模。

图中可能有重边。重边代表不同的边，会产生不同的路径。

### 思路

先看一个可以直接验证想法的朴素解：

@include-code(./brute.cpp, cpp)

暴力可以先求最短距离，再在最短路层次图上计数。因为本题是无权图，正解直接用 BFS。

BFS 有一个关键性质：第一次访问到某个点时，就是从起点到它的最短距离。

设：

```text
dist[x] = 从 1 到 x 的最短距离
ways[x] = 从 1 到 x 的最短路条数
```

从点 `u` 枚举邻点 `v`：

- 如果 `v` 还没访问过，说明第一次找到 `v` 的最短路：

```text
dist[v] = dist[u] + 1
ways[v] = ways[u]
```

- 如果 `dist[v] == dist[u] + 1`，说明又找到了一批到 `v` 的最短路：

```text
ways[v] += ways[u]
```

其它情况不会形成到 `v` 的最短路，不处理。

重边不需要特殊去重。邻接表中有几条边，就会被枚举几次，正好对应多条不同路径。

### 复杂度

BFS 扫描每条边常数次。

时间复杂度为 `O(n+m)`，空间复杂度为 `O(n+m)`。

### 总结

无权图最短路优先考虑 BFS。

最短路计数的核心是只从距离 `d` 的点向距离 `d+1` 的点传递方案数，这样每次累加都对应一条真正的最短路。
