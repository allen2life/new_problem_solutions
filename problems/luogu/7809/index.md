### 题意

给定一个只含 `0/1` 的序列，回答两类区间询问：

- `1 l r`：求 `a[l..r]` 的最长不下降子序列长度
- `2 l r`：求 `a[l..r]` 的最长上升子序列长度

数据范围非常大，`n` 到 `10^6`，询问次数到 `5 * 10^6`，所以每次询问必须非常快。

如果直接在每个区间里做 LIS，哪怕只利用 `01` 串的特殊性，也还是太慢。

#### 1. 最长不下降子序列

在 `01` 串里，不下降子序列只能长成：

```text
若干个 0 + 若干个 1
```

所以只要枚举一个分界点 `k`：

- 从 `l..k` 里取所有 `0`
- 从 `k+1..r` 里取所有 `1`

答案就是：

```text
max( [l..k] 中 0 的个数 + [k+1..r] 中 1 的个数 )
```

设 `pre[i]` 表示前 `i` 个数里 `1` 的个数，那么：

- `[l..k]` 中 `0` 的个数是 `(k-l+1) - (pre[k]-pre[l-1])`
- `[k+1..r]` 中 `1` 的个数是 `pre[r] - pre[k]`

两项相加后得到：

```text
pre[l-1] + pre[r] - l + 1 + (k - 2 * pre[k])
```

对于一次询问，前面那一大段是常数，所以问题就变成：

```text
在区间 [l-1, r] 中求 max( k - 2 * pre[k] )
```

这是静态区间最大值，直接用 ST 表维护即可。

#### 2. 最长上升子序列

严格上升的 `01` 子序列只能是：

```text
0, 1
```

所以答案只可能是：

- `2`：区间里存在一个 `0` 出现在某个 `1` 的前面
- `1`：否则只能取单个元素

于是只要预处理：

- `next_zero[i]`：从 `i` 开始往后第一个 `0` 的位置
- `prev_one[i]`：到 `i` 为止最后一个 `1` 的位置

对于区间 `[l, r]`，如果 `next_zero[l] < prev_one[r]`，说明确实存在 `0` 在 `1` 前面，答案就是 `2`；否则答案是 `1`。

这两个查询都能做到 `O(1)`。

### 复杂度

- 预处理前缀和、`next_zero`、`prev_one`：`O(n)`
- 建 ST 表：`O(n log n)`
- 每次询问：`O(1)`

总时间复杂度 `O(n log n + m)`，空间复杂度 `O(n log n)`。

### 总结

这题的关键不是套普通 LIS 模板，而是抓住 `01` 串的结构非常简单：

- 不下降子序列一定是“前面一段 0，后面一段 1”
- 严格上升子序列最多只有 `0,1` 两个数

把这两个性质用前缀和和位置预处理表达出来以后，问题就能压成常数时间查询。
