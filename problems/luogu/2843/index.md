### 题意

给出一个长度为 `n` 的序列，每个数是一个 `k` 位二进制数。

第 `j` 位为 `1`，表示这个人具有第 `j` 种特性；为 `0` 表示没有。

现在要找一个最长的连续区间，使得这段区间内 `k` 种特性出现的总次数全部相同。

输出这个最长区间的长度。

### 思路

先看最直观的小数据暴力：

@include-code(./brute.cpp, cpp)

`brute.cpp` 先对每一种特性做前缀和，然后枚举所有区间 `[l,r]`，检查这段区间里 `k` 种特性的出现次数是否全部相等。

这个方法能帮助理解题意，但复杂度是 `O(n^2 k)`，显然无法通过 `n = 10^5`。

关键是把条件改写成前缀状态相等。

设 `pre_t[i]` 表示前 `i` 个人中，第 `t` 种特性一共出现了多少次。

如果区间 `[l,r]` 合法，那么：

```text
pre_1[r] - pre_1[l-1]
= pre_2[r] - pre_2[l-1]
= ...
= pre_k[r] - pre_k[l-1]
```

此时最自然的做法，是把所有特性都减去第一种特性，定义：

```text
state(i) = (pre_2[i]-pre_1[i], pre_3[i]-pre_1[i], ..., pre_k[i]-pre_1[i])
```

如果两个前缀位置 `x < y` 满足：

```text
state(x) = state(y)
```

那么对所有 `t >= 2`，都有：

```text
pre_t[y] - pre_1[y] = pre_t[x] - pre_1[x]
```

移项后得到：

```text
pre_t[y] - pre_t[x] = pre_1[y] - pre_1[x]
```

也就是说，区间 `(x,y]` 内每一种特性的出现次数都和第一种特性相同，因此这段区间就是合法区间。

于是原问题变成：

> 找两个相同的前缀状态，使它们之间的距离最大。

实现方式非常直接：

1. 从左到右扫描每个人
2. 更新每种特性的前缀出现次数
3. 构造当前的差分状态 `state(i)`
4. 用哈希表记录这个状态第一次出现的位置
5. 如果当前状态以前出现过，就用当前位置减去第一次出现位置更新答案

别忘了初始全 0 状态出现在位置 `0`，这样从开头开始的合法区间也能被统计进去。

### 复杂度

每扫描一个人：

- 需要枚举 `k` 个二进制位更新前缀计数
- 再构造一次长度为 `k-1` 的差分状态

所以总时间复杂度是：

`O(nk)`

空间复杂度主要来自哈希表中的状态：

`O(nk)`

### 总结

这题的关键不是去直接统计区间，而是把“区间内每种特性出现次数相等”改写成：

- 两个前缀位置的“相对第一种特性的差分状态”完全一样

一旦想到这一步，问题就从区间问题变成了“最长相同前缀状态距离”问题，做法会非常清楚。
