### 题意

给定一个 `n x m` 棋盘，每个格子可以：

- 不染色
- 染成 `1..c` 中某一种颜色

要求：

1. 每一行至少有一个格子被染色
2. 每一列至少有一个格子被染色
3. 每一种颜色都至少出现一次

求满足条件的方案数，对 `1000000007` 取模。

### 思路

先看一个可以直接验证想法的朴素解：

这个暴力会逐格枚举：

- 不染色

最后检查每一行、每一列、每一种颜色是否都满足“至少一次”。它只适合很小的数据，但能很好地说明题意。

正式做法要抓住这题最明显的信号：它有三类“至少一次”限制，分别是行、列、颜色，所以应该想到容斥原理。

#### 固定一个容斥状态后，剩余格子的选择数

这张表描述删掉若干坏对象后的局面：

| 量 | 含义 |
| --- | --- |
| `r` | 还保留了多少行 |
| `s` | 还保留了多少列 |
| `avail - 1` | 还允许出现多少种颜色 |

在这个状态下，只剩 `r x s` 个格子还能自由选择。

而每个这样的格子有：

- `1` 种不染色
- `avail - 1` 种染色方式

所以每个格子一共有 `avail` 种选择，总贡献就是：

`avail^(r*s)`

接下来只要把三层容斥系数乘上去即可：

- 行系数：`(-1)^(n-r) C(n,r)`
- 列系数：`(-1)^(m-s) C(m,s)`
- 颜色系数：`(-1)^(c+1-avail) C(c,c+1-avail)`

因此答案就是三重求和：

`sum_r sum_s sum_avail row_coef[r] * col_coef[s] * color_coef[avail] * avail^(r*s)`

因为 `n,m,c <= 400`，这个三重循环是可以接受的。

代码实现里还做了一个小优化：  
对固定 `avail` 和 `r`，把 `avail^(r*s)` 写成不断乘 `avail^r` 的形式，这样不用在循环里反复快速幂。

### 总结

这题的关键不是设计复杂状态，而是认出：

- 行至少一次
- 列至少一次
- 颜色至少一次

这三类限制都属于“出现性约束”，非常适合直接做容斥。

一旦固定删掉哪些行、列、颜色，剩余格子就完全独立，计数立刻变得简单。
