### 题意

给定一个数列，需要支持三种操作：

- 把区间 `[l, r]` 内每个数乘上 `x`
- 把区间 `[l, r]` 内每个数加上 `x`
- 查询区间 `[l, r]` 的元素和

所有运算都在模 `m` 意义下进行。

### 思路

先看最直观的小数据做法：

@include-code(./brute.cpp, cpp)

`brute.cpp` 直接维护原数组：

- 区间乘时逐个元素乘
- 区间加时逐个元素加
- 查询时逐个累加取模

这个做法容易理解，但复杂度太高。

真正的关键不是“区间求和”本身，而是看清楚区间修改的本质：

- 乘法：`v -> v * x`
- 加法：`v -> v + x`

多次叠加后，它们都可以统一写成一个仿射变换：

`v -> v * mul + add`

于是在线段树节点上只维护区间和 `sum` 即可。

对于一个整段区间：

- 若整段乘上 `x`，则 `sum *= x`
- 若整段加上 `x`，则 `sum += len * x`

难点在于懒标记如何合并。

如果子区间原来已经欠一个操作：

`v -> v * mul + add`

现在父节点又让它整段乘上 `x`，那么新效果应为：

`v -> (v * mul + add) * x = v * (mul * x) + add * x`

所以：

- `lazy_mul *= x`
- `lazy_add *= x`

而整段加上 `x` 时，只需：

- `lazy_add += x`

因此，只要维护：

- `seg_sum[u]`
- `lazy_mul[u]`
- `lazy_add[u]`

并在下传时先传乘法、再传加法，就能正确处理所有操作。

### 复杂度

建树复杂度：

`O(n)`

每次操作复杂度：

`O(log n)`

总时间复杂度：

`O(n + q log n)`

空间复杂度：

`O(n)`

### 总结

这题最核心的一步，是把“区间乘 + 区间加”统一成懒标记里的仿射变换。

只要把标记合成公式推清楚，整题就是一棵标准的懒标记线段树。
