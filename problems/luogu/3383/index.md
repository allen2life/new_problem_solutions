### 题意

给出一个上界 `n` 和 `q` 次询问。  
每次询问给出一个 `k`，要求输出 `1..n` 中第 `k` 小的素数。

### 思路

先看一个可以直接验证想法的朴素解：

从 `2` 开始枚举到 `n`，对每个数做试除法判素。  
如果是素数，就加入数组。  
最后对每个询问输出数组里的第 `k` 个素数。

@include-code(./brute.cpp, cpp)

这个做法最大的问题是建表太慢。  
`n` 可以到 `10^8`，如果每个数都试除，代价会非常大。

这题的正确方向是一次性筛出全部素数，然后查询直接查表。  
题目标题也已经明确提示要用线性筛。

线性筛的核心是：

- 每个合数只由它的最小质因子筛掉一次。

当外层枚举到 `i` 时：

1. 如果 `i` 还没被标成合数，那么 `i` 是素数，把它加入数组；
2. 再用当前已知素数 `p` 去生成 `i * p`；
3. 如果 `i % p == 0`，立刻停止。

下面这个小表格展示了为什么要 `break`：

| 当前 `i` | 枚举到的质数 `p` | 生成的合数 `i*p` | 为什么停止 |
| --- | --- | --- | --- |
| `3` | `2` | `6` | `3 % 2 != 0`，继续 |
| `3` | `3` | `9` | `3 % 3 == 0`，停止 |
| `4` | `2` | `8` | `4 % 2 == 0`，停止 |

表格最后一列表示：一旦 `p` 已经是 `i` 的最小质因子，就不能再继续用更大的素数去生成 `i*p'`，否则会把某些合数重复标记。

这样预处理完成后，`primes[k]` 就是第 `k` 小素数，询问可以直接回答。

### 复杂度

线性筛预处理的时间复杂度是 `O(n)`，每次查询是 `O(1)`。  
空间复杂度是 `O(n)`。

### 总结

这题的重点不在查询，而在于高效建表。

一旦看清“要多次查询第 `k` 小素数”，就应该立刻想到：

- 先筛完整张素数表；
- 再按下标回答。

而线性筛的核心记忆点就是：

- 每个合数只被最小质因子筛掉一次；
- 当 `i % p == 0` 时必须 `break`。
