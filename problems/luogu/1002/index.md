### 题意

棋盘上卒从 $(0,0)$ 出发走到 $(n,m)$，每次只能向下或向右。马在 $(x,y)$，马所在格子和其"日"字跳法能到达的 8 个格子为控制点，卒不能经过。求路径条数。

$n,m \le 20$，答案可能超过 `int` 范围，需要用 `long long`。

### 思路

最直接的想法是 DFS 枚举所有路径，每步向下或向右，遇到终点计数 +1，遇到控制点回溯。

先看一个可以直接验证想法的朴素解：

@include-code(./brute.cpp, cpp)

`brute.cpp` 对小数据正确，但复杂度为 $O(2^{n+m})$，$n=m=20$ 时完全不可行。瓶颈在于反复枚举到达同一格子的不同路径——到达 $(i,j)$ 后，所有从此出发的延伸路径都被重复计算多次。

**关键观察**：卒只能向下或向右，到达 $(i,j)$ 必然来自 $(i-1,j)$ 或 $(i,j-1)$。这满足 DP 的无后效性。

令 $dp[i][j]$ 为从 $(0,0)$ 到 $(i,j)$ 的不经过控制点的路径数：

- 若 $(i,j)$ 是控制点，$dp[i][j] = 0$。
- 否则 $dp[i][j] = dp[i-1][j] + dp[i][j-1]$（上方和左方存在的才加）。
- 边界 $dp[0][0] = 1$。

按行优先遍历一遍即可，每个格子只算一次，复杂度 $O(nm)$，在 $n,m \le 20$ 下轻松通过。

### 复杂度

- 时间：$O(nm)$，标记控制点 $O(1)$，DP 递推 $O(nm)$

### 总结

本题是经典的网格路径计数 DP，核心在于识别"不能走回路 + 只依赖相邻状态"的无后效性。用 DFS 做朴素解做对比，能更直观地理解 DP 从哪里省掉重复计算。
