### 题意

给出若干篇文章，每篇文章由若干单词组成。

接下来查询一些单词，要求输出这个单词在哪些文章中出现过。

### 思路

先看一个可以直接验证想法的朴素解：

@include-code(./brute.cpp, cpp)

`brute.cpp` 用 `map<string, vector<int>>` 直接记录某个单词出现在哪些文章里。

这个思路本身已经可以做出来。

如果从 Trie 角度理解，可以把每个单词插入字典树，并在单词的终止节点维护一个文章编号列表。

这样做的理由很简单：

- 相同单词一定会落到同一个终止节点
- 所以这个终止节点天然可以作为“这个单词的信息存放处”

读入第 `i` 篇文章时，每出现一个单词，就把 `i` 记到这个单词终止节点的列表里。

不过题目只要求知道某篇文章里“是否出现过”这个单词，不关心它出现了多少次，所以同一篇文章里重复出现的单词只能记录一次文章编号。

实现时只要保证：

- 如果当前文章编号和列表最后一个编号相同
- 就不要重复加入

查询时：

- 如果单词在 Trie 中不存在，输出空行
- 如果存在，直接把终止节点上的文章编号列表输出即可

### 复杂度

设所有单词总长度为 `L`。

- 建 Trie：`O(L)`
- 全部查询：`O(L)`

另外还要加上答案输出本身的复杂度。

空间复杂度：

`O(L + 输出总规模)`

### 总结

这题可以看成：

- 单词作为 key
- 文章编号列表作为 value

而 Trie 只是把这个 key 的存储方式从“整串映射”换成了“按字符路径定位”。
