### 题意

给一张连通无向图，每个点有一个“谈话时间” `C_i`，每条边有一个路程时间 `L_j`。

你要删边，尽量少保留道路，但仍然要让所有点连通。之后选择一个起点出发，走遍所有点至少一次，最后回到起点。每次经过一个点，都要再花一次 `C_i` 的时间说话。

问最小总时间是多少。

### 思路

先看一个小数据暴力：

@include-code(./brute.cpp, cpp)

暴力做法是：

1. 枚举所有生成树
2. 再枚举住在哪个点
3. 直接按树上的代价公式计算总时间

这个思路能帮助理解，但边多时肯定不行。

先看“删边尽量多，但还要连通”这一句。能保留的最少边数一定是 `n-1`，所以最后留下来的结构一定是一棵树。

在树上，如果要从某个点出发，走遍所有点，再回到出发点，那么：

- 每条边都必须走两次
  - 一次进子树
  - 一次从子树回来

所以道路贡献固定是：

`2 * 所有保留边长度之和`

再看点权贡献。

设最后保留的是一棵树，起点是 `r`：

- 非起点 `i` 会被经过 `deg(i)` 次
- 起点 `r` 会被经过 `deg(r) + 1` 次
  - 多出来的这一次，就是一开始出发前在家里也要谈一次

所以总谈话时间是：

`sum(deg(i) * C_i) + C_r`

把 `sum(deg(i) * C_i)` 换个角度看：

- 树上每条边 `(u, v)`，都会给 `deg(u)` 和 `deg(v)` 各贡献一次
- 所以所有点权项加起来，等价于把每条边贡献成 `C_u + C_v`

于是整棵树的总代价就是：

`C_r + sum(2 * L(u,v) + C_u + C_v)`

这里 `C_r` 显然应该取全图最小的那个点，因为树一定包含所有点。

剩下的部分就完全变成了最小生成树：

- 把原边权 `L`
- 改成新边权 `2 * L + C_u + C_v`

然后在这张新图上跑一遍 MST 即可。

### 复杂度

设点数为 `n`，边数为 `p`。

Kruskal 的复杂度是：

- 排序 `O(p log p)`
- 并查集合并 `O(p \alpha(n))`

总时间复杂度 `O(p log p)`，空间复杂度 `O(n + p)`。

### 总结

这题难点不在 MST 本身，而在先把“树上闭合走一圈”的总代价拆开。只要看出：

- 边一定走两次
- 点权可以按树边两端分摊

就能把原题规整成一棵带新边权的最小生成树。
