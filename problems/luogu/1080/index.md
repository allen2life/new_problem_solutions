### 题意

国王固定站在最前面，后面有 `n` 位大臣需要重新排序。每个人有左手数 `a` 和右手数 `b`。

某位大臣获得的金币数为：他前面所有人的左手数乘积除以自己的右手数，向下取整。

要求安排大臣顺序，使获得金币最多的大臣的金币数尽量小。

### 思路

先看一个可以直接验证想法的朴素解：

@include-code(./brute.cpp, cpp)

暴力会枚举所有排列，计算每个排列的最大金币数，再取最小值。这个做法只能用于小数据。

关键是相邻交换。

考虑两个相邻大臣 `x=(a_x,b_x)` 和 `y=(a_y,b_y)`，设他们前面所有人的左手乘积为 `P`。

如果顺序是 `x,y`，这两人的最大金币数由下面两个量决定：

```text
P / b_x
P * a_x / b_y
```

如果顺序是 `y,x`，对应为：

```text
P / b_y
P * a_y / b_x
```

公共的 `P` 不影响排序比较。整理后可以得到：当

```text
a_x * b_x <= a_y * b_y
```

时，把 `x` 放在 `y` 前面不会更差。

所以只要按 `a*b` 从小到大排序即可。

排序后从前到后模拟。前缀左手乘积会非常大，需要高精度；但每次只会乘一个小整数、除一个小整数，所以实现高精度乘小整数和除小整数即可。

### 复杂度

排序复杂度为 `O(n log n)`。

设高精度位数为 `D`，每次乘小整数、除小整数为 `O(D)`，总复杂度约为：

```text
O(n log n + nD)
```

空间复杂度为 `O(n + D)`。

### 总结

本题的贪心标准不是单独看左手数或右手数，而是看 `a*b`。

相邻交换证明给出了排序依据；排序完成后，剩下的问题就是高精度模拟前缀乘积和最大金币数。
